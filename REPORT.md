# VisionCLI Project Report

## 1. Project Overview
**VisionCLI** is a terminal-based computer vision interface designed to demonstrate the power of multimodal AI (Gemini) in a command-line environment. It allows users to interact with their camera feed using text-based commands, bridging the gap between traditional CLI workflows and modern computer vision capabilities.

## 2. Technical Architecture
### Frontend
- **React 19**: Core UI framework.
- **Tailwind CSS**: Used for the "Retro-Hacker" terminal aesthetic.
- **Lucide React**: Iconography for the HUD and terminal elements.
- **Motion (Framer Motion)**: Smooth transitions for the preview window and terminal logs.

### AI Engine
- **Gemini 3 Flash**: Used for real-time image analysis. It processes base64-encoded frames captured from the browser's MediaDevices API.
- **System Instructions**: Configured to provide concise, technical, and direct feedback suitable for a CLI environment.

### Key Components
- **Terminal Emulator**: A custom-built React component that handles command parsing, log history, and auto-scrolling.
- **Camera Controller**: Manages the `getUserMedia` stream, frame capture, and canvas processing.

## 3. Features
- **Real-time Camera Integration**: Toggle camera feed via `cam on/off`.
- **Image Scanning**: General description of the environment using `scan`.
- **Object Detection**: Targeted search for specific items using `detect [object]`.
- **OCR (Optical Character Recognition)**: Extraction of text from the camera feed using `text`.
- **HUD Preview**: A toggleable visual feed with a technical HUD overlay.

## 4. Implementation Details
The application captures a frame from the `<video>` element using a hidden `<canvas>`, converts it to a JPEG base64 string, and sends it to the Gemini API along with the user's command. The response is then streamed back into the terminal log.

## 5. Usage Instructions
1. Type `cam on` to initialize the camera.
2. Type `scan` to get an overview of your surroundings.
3. Type `detect cat` to look for a cat in the frame.
4. Type `text` to read any documents or signs in view.
5. Type `help` for a full list of commands.

## 6. Future Enhancements
- **Batch Processing**: Ability to queue multiple frames for temporal analysis.
- **Local Storage**: Saving scan results to a local "database" (JSON file).
- **Custom Models**: Integration with specialized edge-detection or depth-sensing APIs.
